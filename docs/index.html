7. Что записывается в лог и чем уровень ERROR отличается от WARNING?

В лог записываются события работы программы (время, модуль, сообщение). WARNING указывает на потенциальную проблему, которая не мешает работе. ERROR сообщает о сбое конкретной функции, при котором программа продолжает выполняться.

8. Зачем в logging.basicConfig указывают сразу два обработчика – FileHandler и StreamHandler?

Это позволяет одновременно выводить логи в консоль (для оперативного контроля разработчиком через StreamHandler) и сохранять их в файл (для последующего анализа через FileHandler).

9. Что делает блок finally и почему он важен при работе с базой данных?

Блок finally выполняется всегда, независимо от ошибок. Он критически важен для безопасного закрытия соединения с БД (tasks.db), чтобы предотвратить утечки памяти и блокировку файла базы.

10. Почему имя пакета в pip install может отличаться от имени в import?

Потому что pip install устанавливает название дистрибутива в репозитории PyPI, а import подключает конкретное имя модуля или папки внутри кода. Это решение автора библиотеки (например, pip install pillow, но import PIL).

11. Что такое миграция базы данных? В какой ситуации она нужна?

Это обновление структуры таблиц (добавление колонок, изменение типов) без потери существующих данных. Она нужна при развитии проекта, когда требуется изменить схему БД в файле tasks.db.

12. Какой SQL-командой в SQLite добавляют новую колонку в таблицу?

Используется команда ALTER TABLE. Пример: ALTER TABLE tasks ADD COLUMN deadline TEXT;

13. Что должен содержать файл README.txt при передаче проекта?

Файл должен содержать краткое описание проекта, список файлов, инструкции по установке зависимостей, способ запуска и информацию о том, где найти результаты (отчеты и логи).
